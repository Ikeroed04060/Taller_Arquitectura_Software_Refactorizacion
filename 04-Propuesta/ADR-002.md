# ADR-002: Usar JPA y Spring Data en lugar de SQL Directo

## Contexto
El uso de concatenación SQL en el código del controlador genera vulnerabilidades, como el **SQL Injection**, y dificulta la gestión de consultas en el sistema.

## Decisión
Implementar **JPA (Java Persistence API)** y **Spring Data** para manejar las consultas a la base de datos. Este enfoque evitará el uso de consultas SQL directas, lo que mejorará la seguridad y facilitará la persistencia de datos mediante el uso de repositorios.

## Consecuencias
- **Ventajas**: Mejora la seguridad al evitar inyecciones SQL. Facilita el mantenimiento y la escalabilidad del código al usar un enfoque más estructurado y flexible para interactuar con la base de datos.
- **Desventajas**: Requiere un aprendizaje adicional de **Spring Data** y la configuración inicial de **JPA**, lo que podría generar una curva de aprendizaje en el equipo.
