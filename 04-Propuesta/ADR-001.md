# ADR-001: Adoptar Arquitectura en Capas (Layered Architecture)

## Contexto
El sistema actual tiene un controlador monolítico con múltiples responsabilidades, lo que viola el principio **Single Responsibility Principle (SRP)** y dificulta el mantenimiento y las pruebas unitarias.

## Decisión
Implementar una arquitectura en capas que divida el sistema en tres capas claramente definidas:
- **Controller**: Encargado de recibir las solicitudes HTTP.
- **Service**: Contiene la lógica de negocio.
- **Repository**: Se encarga de la persistencia de los datos.

Esta estructura facilita el mantenimiento, las pruebas unitarias y la escalabilidad, y permite agregar nuevas funcionalidades sin modificar el sistema existente.

## Consecuencias
- **Ventajas**: Mejora la organización y mantenibilidad del código. Facilita las pruebas unitarias y futuras ampliaciones.
- **Desventajas**: Incremento en el número de clases y la complejidad inicial al separar las responsabilidades.
