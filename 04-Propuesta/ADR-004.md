# ADR-004: Externalizar la Configuración

## Contexto
El sistema actual tiene configuraciones **hardcodeadas** en el código (por ejemplo, URL de la API). Esto hace que el mantenimiento sea más difícil, especialmente al pasar a diferentes entornos de desarrollo, pruebas y producción.

## Decisión
Utilizar archivos de **configuración externa** (por ejemplo, `application.yml` o variables de entorno) para almacenar las configuraciones del sistema (como las URLs de las APIs y las credenciales de la base de datos). Además, en el futuro, considerar un **Config Server** para centralizar y gestionar las configuraciones de forma dinámica.

## Consecuencias
- **Ventajas**: Mejora la flexibilidad y la seguridad, ya que las configuraciones no estarán hardcodeadas en el código. Facilita la migración a diferentes entornos y reduce el riesgo de errores al hacer cambios.
- **Desventajas**: Requiere esfuerzo adicional para establecer y gestionar un sistema centralizado de configuraciones.
