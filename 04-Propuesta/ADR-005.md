# ADR-005: Uso de Docker para Despliegue y Migración a Microservicios

## Contexto
El sistema monolítico debe ser fácil de desplegar y migrar a microservicios en el futuro. El uso de Docker ayudará a contenerizar la aplicación y facilitará una transición más sencilla a microservicios si es necesario.

## Decisión
Utilizar **Docker** para contenerizar la aplicación desde el principio. Esto no solo facilitará el despliegue del sistema monolítico, sino que también hará más fácil la migración a microservicios, ya que cada servicio podrá ser ejecutado como un contenedor independiente.

## Consecuencias
- **Ventajas**: Facilita el despliegue y la migración a microservicios. Mejora la consistencia y la portabilidad entre entornos de desarrollo y producción.
- **Desventajas**: Requiere configuración y mantenimiento de los contenedores Docker, lo que puede aumentar la complejidad inicial.
